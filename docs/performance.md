# Performance

## 現在の対策

### ツリー遅延読み込み

- 展開時のみ子要素を取得
- `node_modules`、`.git`、`.cache` を非表示
- 大きいディレクトリは警告表示
- TREE は仮想スクロールを使わず、現在は全件描画で見切れ防止を優先している

### ログ末尾読み込み

- `history.jsonl` を全件読まない
- `createReadStream` で末尾側だけを取得
- 末尾 1000 行のみ集計

### ファイルサイズ制限

- 5MB 以上の非 PDF はプレビュー対象外
- バイナリファイルは読まない
- CSV は 1000 行までに制限

### ファイル監視の限定

- `fs.watch` は現在表示中ファイルだけに限定
- 多重 watch を防ぐため main 側で常に 1 本のみ保持

## 軽量化の理由

- このアプリは CLI 補助ツールなので、主処理を邪魔しないことが優先
- 深い走査や大量読込を避けて、体感速度を落とさない方針

## 今後の改善余地

- ファイル検索のインデックス化
- JSON / CSV のストリーム表示
- PDF 以外の大容量ファイル向け部分読込
- 設定の永続化項目追加
